# 言語クラウドソーシング — ユーザーガイド (日本語)

このガイドでは、**言語クラウドソーシング**プラグイン (`local_langcrowd`) の使い方を、
翻訳に投票する参加者の立場と、翻訳を管理・エクスポートする管理者の立場の両方から説明します。

---

## 1. このプラグインの機能

言語クラウドソーシングを使うと、ユーザーは Moodle を離れることなく Moodle の
インターフェース翻訳を改善できます。有効にすると、フローティングの
**翻訳を改善する**ボタンが表示されます。これをオンにすると、画面上の短いテキストの
多くに小さなボタンが 2 つ表示されます。つまり、あなたが手伝うことを選ぶまでは何も変わりません。

- **✓ (チェック)** — 「この翻訳は良い」。十分な人数が承認すると、その文字列はロックされ、
  すべての人に提供されます。
- **✗ (バツ)** — 「もっと良くできる」。代わりの訳を入力するよう求められ、
  それは管理者のレビューに送られます。

あなたが手伝う対象の言語は、現在の Moodle インターフェース言語そのものです。
言語を切り替えると、投票する翻訳の対象も切り替わります。

---

## 2. 参加者の方へ

### 翻訳モードをオンにする

ログインしている必要があります (ゲストは投票できません)。右下隅の
**翻訳を改善する**ボタンをクリックして翻訳モードをオンにします。もう一度クリックすると
オフになります。この選択はページを移動しても記憶されます。ボタンがまったく表示されない場合、
サイトでクラウドソーシングが特定のロール・言語・コンポーネントに限定されている可能性があります。
管理者にお問い合わせください。

翻訳モードがオンになると、テキストにマウスを合わせたときにボタンが表示されます
(サイトの設定によっては常に表示され、タッチスクリーンでは常に表示されます)。

### 翻訳を承認する

語句にマウスを合わせて緑色の **✓** をクリックします。ツールチップには、その語句が
ロックにどれだけ近いか (例: 「5/10 承認」) が表示されます。あなたの投票が記録され、
その語句のボタンは消えます。気が変わりましたか? 数秒間だけ **元に戻す** リンクが表示され、
投票を取り消すことができます。

### より良い翻訳を提案する

赤色の **✗** をクリックします。英語の原文と現在のテキスト、そしてあなたの提案を入力する
ボックスを表示したダイアログが開きます。改善した翻訳を入力して **提案を送信** を
クリックします。あなたの提案は管理者に送られ、同時に「要改善」の投票が記録されます。
送信せずに閉じるには **Esc** または **キャンセル** を押します。

### その後の流れ

- サイトの投票しきい値に達して承認された文字列は自動的に **ロック** され、
  すべての人に表示されます。
- あなたの提案は管理者がレビューし、承認する (直ちに反映) か却下します。

一度投票した語句のボタンは、二度と表示されません。

## 3. 管理者の方へ

すべての管理機能は **サイト管理 → 言語 → 言語クラウドソーシング** にあります。

### 3.1 初回設定

1. `config.php` に 1 行追加します (末尾の `require_once` より前):

   ```php
   $CFG->customstringmanager = '\local_langcrowd\string_manager';
   ```

   設定ページでは、これが有効なときに緑色のバナーが表示され、未設定のときは警告が表示されます。

2. **設定**に移動し、**クラウドソーシングを有効にする**にチェックを入れます。

### 3.2 設定リファレンス

| 設定 | 説明 |
|---|---|
| クラウドソーシングを有効にする | メインのオン/オフスイッチ。 |
| ナビゲーションバーに管理リンクを表示 | トップナビゲーションにプラグインへのショートカットを追加します (管理者のみ)。 |
| 管理者の承認投票で即座にロック | オンにすると、サイト管理者による承認投票が即座に文字列をロックします。 |
| 承認しきい値 | 文字列をロックするために必要な承認投票の数 (デフォルト 10)。 |
| ページごとの最大文字列数 | 1 ページで注釈を付ける文字列数の上限 (デフォルト 5000)。 |
| ボタンの表示モード | ボタンをホバー時のみ表示するか、常に表示するか (タッチデバイスは常に「常に表示」)。 |
| 文字列のハイライト色 | ホバー時に語句の周囲に描かれるアウトラインの色。 |
| 投票を許可するロール | 投票を特定のロールに制限します。空欄 = ログイン済みの全ユーザー。**サーバー側で強制されます。** |
| クラウドソーシングを有効にする言語 | オーバーレイを特定の言語に制限します。空欄 = すべて。 |
| クラウドソーシングを有効にするコンポーネント | オーバーレイを特定のコンポーネント (例: `mod_forum`、`core`) に制限します。空欄 = すべて。リストはこれまでに検出された文字列から作成されます。クリアすると新しいコンポーネントが再び検出されます。 |

### 3.3 概要

**概要**ページはプラグインのダッシュボードです。保留中・適用済み・ロック済みの文字列数、
追跡中の文字列の総数、待機中の提案数に加えて、まだ投票を受け付けている最も投票の多い文字列と
最新の提案を、レポートやエクスポート機能への簡単なリンクとともに表示します。

### 3.4 投票レポート

投票アクティビティのあるすべての文字列を、3 つのステータス別に一覧表示します。

- **保留中** (グレー) — 投票受付中、まだ昇格していません。
- **適用済み** (青) — 承認された提案がライブで提供され、引き続き投票を受け付けています。
- **ロック済み** (緑) — 確定し、有効な翻訳として提供されています。

テーブルには現在の翻訳の横に英語の原文が表示されます。言語・コンポーネント・ステータスで
フィルタでき、**投票のない文字列を含める**にチェックするとすべて表示され、
列見出しをクリックすると並べ替えられます。各行には操作ボタンがあり (確認ダイアログ付き)、
チェックボックスを選択してテーブル下部の操作バーを使えば複数行を一括操作できます。

- **ロック** — しきい値を待たずに保留中の文字列を直ちにロックします。
- **削除** — ロック済み/適用済みの文字列を保留中に戻し、投票をリセットします。

### 3.5 ユーザーの提案

ユーザーからの保留中の提案を、英語の原文・現在の翻訳・提案・投稿者とともに一覧表示します。
各操作は行ごと、または一括で適用できます (チェックボックスを選択して操作バーを使用):

- **承認** — 提案を有効な翻訳にしてロックします (投票をリセット)。
- **言語パックに適用** — 直ちにライブで提供しつつ、コミュニティの投票を受け付けたままにします。
  しきい値に達すると自動的にロックされます。
- **却下** — 現在の翻訳を変えずに提案を破棄します。

### 3.6 言語パックのエクスポート

**言語パックのエクスポート**を開き、単一の言語または**すべての言語**を選択し、
必要に応じてコンポーネントでフィルタし (*クラウドソーシングを有効にするコンポーネント*
設定で選択したコンポーネントがあらかじめ選択されています)、**ロックされた文字列のみ**または
**翻訳のあるすべての文字列**を選び、**言語パックをダウンロード**をクリックします。
標準的な Moodle 言語パックの `.zip` が得られ、**サイト管理 → 言語 → 言語パック**から
インストールするか、Moodle の `lang/` ディレクトリに展開できます。

---

## 4. ロックと提供のしくみ

- ロック済みおよび適用済みの翻訳は、プラグインによって直ちに提供されます。自分のサイトで
  反映させるためにエクスポートやパックのインストールを行う必要はありません。
- エクスポートが必要になるのは、翻訳を別のサイトに移したり、上流に貢献したりする場合だけです。

---

## 5. プライバシー

このプラグインは、あなたが投じた投票と送信した提案をユーザーごとに保存します。
このデータは Moodle のプライバシー (GDPR) ツールの対象です。データエクスポートに含まれ、
データ削除リクエストによって削除されます。
