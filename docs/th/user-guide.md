# การแปลภาษาแบบร่วมสร้าง — คู่มือผู้ใช้ (ภาษาไทย)

คู่มือนี้อธิบายวิธีใช้ปลั๊กอิน **การแปลภาษาแบบร่วมสร้าง** (`local_langcrowd`)
ทั้งในฐานะผู้ร่วมลงคะแนนการแปล และในฐานะผู้ดูแลระบบที่จัดการและส่งออกการแปล

---

## 1. ปลั๊กอินนี้ทำอะไร

การแปลภาษาแบบร่วมสร้างช่วยให้ผู้ใช้ปรับปรุงการแปลอินเทอร์เฟซของ Moodle
ได้โดยไม่ต้องออกจาก Moodle เมื่อเปิดใช้งาน จะมีปุ่มลอย **ปรับปรุงการแปล** ปรากฏขึ้น
เมื่อคุณเปิดใช้งาน ข้อความสั้น ๆ ส่วนใหญ่บนหน้าจอจะมีปุ่มเล็ก ๆ สองปุ่ม —
ดังนั้นจะไม่มีอะไรเปลี่ยนแปลงจนกว่าคุณจะเลือกช่วยแปล:

- **✓ (ถูก)** — "การแปลนี้ดีแล้ว" เมื่อมีคนอนุมัติสตริงมากพอ
  สตริงนั้นจะถูกล็อกและแสดงให้ทุกคนเห็น
- **✗ (กากบาท)** — "น่าจะดีกว่านี้ได้" ระบบจะให้คุณพิมพ์คำแปลใหม่
  ซึ่งจะถูกส่งไปให้ผู้ดูแลระบบตรวจสอบ

ภาษาที่คุณช่วยแปลคือภาษาอินเทอร์เฟซ Moodle ปัจจุบันของคุณ
เมื่อเปลี่ยนภาษา คุณก็จะเปลี่ยนการแปลที่กำลังลงคะแนนด้วย

---

## 2. สำหรับผู้ร่วมลงคะแนน

### การเปิดโหมดแปล

คุณต้องเข้าสู่ระบบก่อน (ผู้เยี่ยมชมลงคะแนนไม่ได้) คลิกปุ่ม **ปรับปรุงการแปล**
ที่มุมขวาล่างเพื่อเปิดโหมดแปล คลิกอีกครั้งเพื่อปิด ระบบจะจดจำการเลือกของคุณ
ขณะที่คุณย้ายไปมาระหว่างหน้า หากคุณไม่เห็นปุ่มเลย การร่วมสร้างอาจถูกจำกัดเฉพาะ
บางบทบาท บางภาษา หรือบางคอมโพเนนต์ โปรดสอบถามผู้ดูแลระบบ

เมื่อเปิดโหมดแปลแล้ว ปุ่มจะปรากฏเมื่อคุณเลื่อนเมาส์ไปเหนือข้อความ
(หรือแสดงตลอดเวลาหากเว็บไซต์ตั้งค่าไว้เช่นนั้น และบนหน้าจอสัมผัสจะแสดงตลอดเวลาเสมอ)

### การอนุมัติการแปล

เลื่อนเมาส์ไปเหนือวลีแล้วคลิกปุ่มสีเขียว **✓** คำแนะนำเครื่องมือจะแสดงว่า
วลีนั้นใกล้ถูกล็อกแค่ไหน (เช่น "5/10 การอนุมัติ") คะแนนของคุณจะถูกบันทึก
และปุ่มของวลีนั้นจะหายไป เปลี่ยนใจหรือไม่? ลิงก์ **เลิกทำ** จะปรากฏสองสามวินาที
เพื่อให้คุณถอนคะแนนได้

### การเสนอคำแปลที่ดีกว่า

คลิกปุ่มสีแดง **✗** กล่องจะเปิดขึ้นแสดงต้นฉบับภาษาอังกฤษและข้อความปัจจุบัน
พร้อมช่องสำหรับข้อเสนอของคุณ พิมพ์คำแปลที่ปรับปรุงแล้วคลิก **ส่งข้อเสนอแนะ**
ข้อเสนอจะถูกส่งไปให้ผู้ดูแลระบบ พร้อมทั้งบันทึกคะแนน "ควรปรับปรุง" ไปด้วย
กด **Esc** หรือ **ยกเลิก** เพื่อปิดโดยไม่ส่ง

### หลังจากนั้นเกิดอะไรขึ้น

- สตริงที่ได้รับการอนุมัติถึงเกณฑ์คะแนนของเว็บไซต์จะถูก **ล็อก**
  และแสดงให้ทุกคนเห็นโดยอัตโนมัติ
- ข้อเสนอของคุณจะได้รับการตรวจสอบโดยผู้ดูแลระบบ ซึ่งอาจยอมรับ
  (ทำให้ใช้งานทันที) หรือปฏิเสธ

เมื่อคุณลงคะแนนวลีใดแล้ว คุณจะไม่เห็นปุ่มของวลีนั้นอีก

---

## 3. สำหรับผู้ดูแลระบบ

การจัดการทั้งหมดอยู่ที่ **การดูแลระบบเว็บไซต์ → ภาษา → การแปลภาษาแบบร่วมสร้าง**

### 3.1 การตั้งค่าครั้งแรก

1. เพิ่มบรรทัดหนึ่งใน `config.php` (ก่อน `require_once` บรรทัดสุดท้าย):

   ```php
   $CFG->customstringmanager = '\local_langcrowd\string_manager';
   ```

   หน้าการตั้งค่าจะแสดงแถบสีเขียวเมื่อทำงานอยู่ และแสดงคำเตือนหากยังไม่ได้ตั้งค่า

2. ไปที่ **การตั้งค่า** แล้วติ๊ก **เปิดใช้งานการแปลแบบร่วมสร้าง**

### 3.2 รายการการตั้งค่า

| การตั้งค่า | ความหมาย |
|---|---|
| เปิดใช้งานการแปลแบบร่วมสร้าง | สวิตช์เปิด/ปิดหลัก |
| ล็อกโหมดแปลให้เปิดตลอด | ซ่อนปุ่มสลับ "ปรับปรุงการแปล" แบบลอย และเปิดอินเทอร์เฟซไว้ตลอดเวลาสำหรับทุกคนที่ใช้งานได้ (แทนที่จะให้ผู้ใช้แต่ละคนเปิดเอง) |
| แสดงลิงก์ผู้ดูแลระบบในแถบนำทาง | เพิ่มทางลัดไปยังปลั๊กอินในแถบนำทางด้านบน (เฉพาะผู้ดูแลระบบ) |
| การโหวตอนุมัติของผู้ดูแลระบบล็อกทันที | หากเปิด การโหวตอนุมัติของผู้ดูแลระบบจะล็อกสตริงทันที |
| เกณฑ์การอนุมัติ | จำนวนคะแนนอนุมัติที่ล็อกสตริง (ค่าเริ่มต้น 10) |
| จำนวนสตริงสูงสุดต่อหน้า | ขีดจำกัดสตริงที่มีปุ่มต่อหน้า (ค่าเริ่มต้น 5000) |
| โหมดการแสดงปุ่ม | แสดงปุ่มเมื่อเลื่อนเมาส์เท่านั้น หรือแสดงตลอดเวลา (อุปกรณ์สัมผัสจะแสดงตลอดเวลาเสมอ) |
| สีไฮไลต์สตริง | สีของเส้นขอบที่วาดรอบวลีเมื่อเลื่อนเมาส์ |
| บทบาทที่ได้รับอนุญาตให้ลงคะแนน | จำกัดการลงคะแนนเฉพาะบางบทบาท เว้นว่าง = ผู้ใช้ที่เข้าสู่ระบบทุกคน **บังคับใช้ที่ฝั่งเซิร์ฟเวอร์** |
| ภาษาที่เปิดใช้งานการแปลแบบร่วมสร้าง | จำกัดเฉพาะบางภาษา เว้นว่าง = ทุกภาษา |
| คอมโพเนนต์ที่เปิดใช้งานการแปลแบบร่วมสร้าง | จำกัดเฉพาะบางคอมโพเนนต์ (เช่น `mod_forum`, `core`) เว้นว่าง = ทั้งหมด รายการสร้างจากสตริงที่พบมาแล้ว ล้างเพื่อให้ค้นพบคอมโพเนนต์ใหม่อีกครั้ง |

### 3.3 ภาพรวม

หน้า **ภาพรวม** คือแดชบอร์ดของปลั๊กอิน แสดงจำนวนสตริงที่รอดำเนินการ ส่งเข้าชุดภาษา
และล็อกแล้ว จำนวนสตริงที่ติดตามทั้งหมด และจำนวนข้อเสนอแนะที่รออยู่ พร้อมทั้ง
สตริงที่มีคะแนนมากที่สุดที่ยังเปิดให้ลงคะแนนและข้อเสนอแนะล่าสุด
พร้อมลิงก์ด่วนไปยังรายงานและตัวส่งออก

### 3.4 รายงานการลงคะแนน

แสดงสตริงทุกตัวที่มีกิจกรรมการลงคะแนน แบ่งเป็นสามสถานะ:

- **รอดำเนินการ** (เทา) — เปิดให้ลงคะแนน ยังไม่ได้ส่งเสริม
- **ส่งเข้าชุดภาษาแล้ว** (น้ำเงิน) — ข้อเสนอที่ยอมรับแล้วและใช้งานอยู่ แต่ยังเปิดให้ลงคะแนน
- **ล็อกแล้ว** (เขียว) — ตัดสินแล้วและใช้เป็นการแปลที่ใช้งานอยู่

ตารางจะแสดงต้นฉบับภาษาอังกฤษถัดจากการแปลปัจจุบัน คุณสามารถกรองตามภาษา
คอมโพเนนต์ และสถานะ ติ๊ก **รวมสตริงที่ไม่มีคะแนน** เพื่อดูทั้งหมด
และคลิกหัวคอลัมน์เพื่อจัดเรียง แต่ละแถวมีปุ่มการดำเนินการ (พร้อมการยืนยัน)
และคุณสามารถดำเนินการหลายแถวพร้อมกันได้โดยติ๊กช่องทำเครื่องหมาย
แล้วใช้แถบการดำเนินการใต้ตาราง:

- **ล็อก** — ล็อกสตริงที่รอดำเนินการทันทีโดยไม่ต้องรอเกณฑ์คะแนน
- **นำออก** — เปลี่ยนสตริงที่ล็อก/ส่งเข้าชุดภาษากลับเป็นรอดำเนินการและรีเซ็ตคะแนน

### 3.5 ข้อเสนอแนะของผู้ใช้

แสดงข้อเสนอที่รอดำเนินการจากผู้ใช้ พร้อมต้นฉบับภาษาอังกฤษ การแปลปัจจุบัน
ข้อเสนอ และผู้ส่ง แต่ละการดำเนินการใช้ได้ทั้งรายแถวหรือแบบกลุ่ม
(ติ๊กช่องทำเครื่องหมายแล้วใช้แถบการดำเนินการ):

- **อนุมัติ** — ทำให้ข้อเสนอเป็นการแปลที่ใช้งานและล็อกทันที (รีเซ็ตคะแนน)
- **ส่งเข้าชุดภาษา** — ใช้งานทันทีแต่ยังเปิดให้ชุมชนลงคะแนน
  และจะล็อกอัตโนมัติเมื่อถึงเกณฑ์
- **ปฏิเสธ** — ยกเลิกข้อเสนอ โดยไม่เปลี่ยนการแปลปัจจุบัน

### 3.6 การส่งออกชุดภาษา

เปิด **ส่งออกชุดภาษา** เลือกภาษาเดียวหรือ **ทุกภาษา**
กรองตามคอมโพเนนต์ได้ตามต้องการ (คอมโพเนนต์ที่เลือกในการตั้งค่า
*คอมโพเนนต์ที่เปิดใช้งานการแปลแบบร่วมสร้าง* จะถูกเลือกไว้ล่วงหน้าให้)
เลือก **เฉพาะสตริงที่ล็อกแล้ว** หรือ **สตริงทั้งหมดที่มีการแปล**
แล้วคลิก **ดาวน์โหลดชุดภาษา** คุณจะได้ไฟล์ `.zip` ชุดภาษา Moodle มาตรฐาน
ที่ติดตั้งได้ที่ **การดูแลระบบเว็บไซต์ → ภาษา → ชุดภาษา**
หรือแตกไฟล์ลงในโฟลเดอร์ `lang/` ของ Moodle

---

## 4. การล็อกและการแสดงผลทำงานอย่างไร

- การแปลที่ล็อกและส่งเข้าชุดภาษาจะแสดงผลทันทีโดยปลั๊กอิน คุณไม่ต้อง
  ส่งออกหรือติดตั้งชุดภาษาเพื่อให้มีผลบนเว็บไซต์ของคุณเอง
- การส่งออกจำเป็นเฉพาะเมื่อคุณต้องการย้ายการแปลไปยังเว็บไซต์อื่น
  หรือส่งกลับไปยังต้นทาง

---

## 5. ความเป็นส่วนตัว

ปลั๊กอินจัดเก็บคะแนนที่คุณลงและข้อเสนอที่คุณส่งเป็นรายบุคคล ข้อมูลนี้
อยู่ภายใต้เครื่องมือความเป็นส่วนตัว (GDPR) ของ Moodle: จะรวมอยู่ในการส่งออกข้อมูล
และถูกลบเมื่อมีคำขอลบข้อมูล
